# 3.0 — Things You Might Run Into

Add-on to [`03.1-organizations-and-accounts.md`](03-organizations-and-accounts.md). Real snags people hit while creating accounts and OUs, with the fix for each.

**"This email already exists" when creating an account.**
Every AWS account needs its own unique email, including your management account. You cannot reuse the email you signed up with. Use `+` aliases (`you+resumeprod@yourcompany.org`, `you+resumedev@...`) or dedicated addresses (`aws-resume-prod@yourcompany.org`) so each is unique but still lands in your inbox. If a half-finished attempt already grabbed an alias, just bump it (`+resumeprod2`). Make sure it's an inbox you can actually receive mail at, since AWS sends a notice there.

**The "Actions" menu items are greyed out.**
The menu is context-sensitive: it acts on whatever is *checked*. To **create an OU**, check the box next to **Root** first, then Actions → Create new. To **rename or move an account**, check exactly **one account** (not Root, not the OU, not several at once). If nothing is selected, everything is disabled.

**"Move" won't work / Destination says "no child organizational units."**
You can only move an account *into a folder that already exists*. If there's no OU yet, there's nowhere to move it. **Create the OU first** (check Root → Actions → Create new → name it), then go back and move the account into it.

**You created an account and nothing popped up.**
That's normal. Account creation happens quietly in the background (usually under a minute) with no confirmation window and no redirect. Refresh the Organizations page and the new account appears under Root. You'll also get an email at the address you used. If it's been several minutes with nothing, check that inbox, the creation may have failed and need a retry.

**You can't rename an account from the Organizations screen.**
AWS Organizations has no "Rename" for accounts in that menu (the greyed-out "Rename" there is for the *OU*, not the account). To rename an *account*, sign into that account and change it under the account menu → **Account** settings → Account name. This is far easier *after* you set up Identity Center (doc 04), which gives you simple access into each account. Note: account names are **cosmetic** — the unique emails and account IDs already tell your accounts apart, so don't let a rename block you. Leave them and rename later.

**Don't delete everything to "start over."**
Closing a member account does **not** remove it right away — it enters a **90-day suspended state** and keeps sitting in your org the whole time, with its email still tied up. So deleting to fix something cosmetic (like duplicate display names) costs you hours now and a 90-day cleanup, for no real gain. Only start over if something is *structurally* wrong (wrong/inaccessible emails, accounts in the wrong org). A correct structure with imperfect names is fine, keep it.

**A banner says "1 request to create an AWS account has failed in the last 90 days."**
Usually harmless. It's just a record of an earlier failed attempt (often the "email already exists" one). Nothing to fix.

**You can't put an account "inside" another account.**
Accounts don't nest. The only thing that groups accounts is an **OU (a folder)**. Your management account stays on its own under Root; workload accounts go into OUs next to it, never inside it.

**Org actions seem to do nothing / you lack permission.**
All Organizations actions must be done from the **management account**. If you switched into a member account, switch back to the management account to manage the org.
